## App configuration

The mobile app is configured in `capacitor.config.ts`:

```typescript
const config: CapacitorConfig = {
  appId: 'com.nobadfonts.app',
  appName: 'NoBadFonts',
  webDir: 'dist'
};
```

- **App ID**: `com.nobadfonts.app` (unique Android package identifier)
- **App Name**: "NoBadFonts" (shown on home screen and app switcher)
- **Web Directory**: `dist` (compiled React app output)

## Features

The mobile app includes all core features of the web version:

### Font catalog

- **Responsive grid**: Adapts from 1-2 columns based on device width
- **Touch-optimized cards**: Large tap targets for font cards
- **Pull-to-refresh**: Swipe down to reload the catalog
- **Infinite scroll**: Progressive loading as you scroll
- **Search and filters**: Full filtering capability

### Font tester

- **Touch controls**: Sliders optimized for thumb interaction
- **Keyboard input**: Native keyboard for text entry
- **Color pickers**: Native color picker dialog
- **Variant selector**: Dropdown menu for weight/style selection
- **OpenType features**: Collapsible panel saves screen space

### Font pairing

- **Compact controls**: Stacked layout for mobile screens
- **Inline editing**: Tap text blocks to edit directly
- **Font picker modal**: Full-screen modal for font selection
- **Background color**: Quick swatches for background changes
- **Gesture controls**: Swipe to add/remove blocks (planned)

### Font details

- **Gallery swiper**: Horizontal scroll through preview images
- **Download**: One-tap download to device storage
- **Share**: Native share sheet integration
- **Favorite**: Sync with web account

## Mobile-specific optimizations

### Performance

<Accordion title="Lazy loading">
  Images and fonts load on demand as you scroll to conserve bandwidth and memory.
</Accordion>

<Accordion title="Caching">
  Capacitor's built-in HTTP cache stores font files locally after first download.
</Accordion>

<Accordion title="Reduced animations">
  Respects prefers-reduced-motion for users with motion sensitivity.
</Accordion>

<Accordion title="Offline fallbacks">
  Core UI works offline with cached data (full offline mode coming soon).
</Accordion>

### UI adaptations

**Navigation**:
- Bottom navigation bar for key sections
- Large tap targets (48x48px minimum)
- Native Android back button support

**Typography**:
- Larger base font size (16px vs 14px)
- Increased line height for readability
- Reduced letter spacing on small screens

**Layout**:
- Single column on phones (&lt;640px)
- Two columns on large phones and small tablets (640-1024px)
- Full desktop layout on tablets (1024px+)

## Technical details

### Web view

The app uses **Android System WebView** to render the React app:
- Chromium-based rendering engine
- Full ES6+ JavaScript support
- CSS Grid, Flexbox, and modern CSS
- HTML5 APIs (Canvas, Web Fonts, etc.)

### Bridge

Capacitor's JavaScript bridge connects web code to native APIs:
- Bidirectional communication (web ↔ native)
- Async/await pattern for native calls
- Type-safe plugin APIs
- Plugin extensibility

### Build

The build process:
1. Vite compiles React app to static HTML/CSS/JS
2. Capacitor copies build to `android/app/src/main/assets/public/`
3. Android WebView loads `index.html` on app launch
4. Capacitor injects bridge JavaScript
5. App initializes and connects to Supabase

## Troubleshooting

<AccordionGroup>
  <Accordion title="App won't install">
    - Check "Install from unknown sources" is enabled
    - Verify the APK isn't corrupted (re-download)
    - Ensure device meets minimum Android version (5.0+)
  </Accordion>

  <Accordion title="Blank screen on launch">
    - Check internet connection (app needs initial data fetch)
    - Clear app data in Android settings
    - Reinstall the app
  </Accordion>

  <Accordion title="Fonts not loading">
    - Verify internet connection
    - Check CORS headers on font URLs
    - Try refreshing the catalog
  </Accordion>

  <Accordion title="Slow performance">
    - Close background apps to free memory
    - Clear app cache
    - Disable animation effects in system settings
  </Accordion>
</AccordionGroup>
